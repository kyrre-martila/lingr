# Run 9 — Discovery Polish Review

## Before → after philosophy
Before: Discovery could still feel like a lightweight profile card flow with pacing hints.
After: Discovery is intentionally framed as a calm introduction experience with one person at a time, Glimps-first storytelling, and quiet emotional context.

## Lingr alignment review
- Visual tone now better aligns with onboarding/chat through softer spacing, quieter surfaces, and less transactional metadata exposure.
- Action language is normalized to calm vocabulary: `Send spark`, `Not now`.
- Region unavailable state uses warm explanatory phrasing and a soft placeholder CTA path.
- Spark success remains subtle and non-gamified.

## UX tradeoffs
- Reduced explicit pacing metadata lowers performance-like pressure, but also reduces analytical clarity for users who prefer counts.
- Glimps-first layout improves emotional context at the cost of showing less profile breadth on first view.
- Soft state transitions prioritize calmness over speed/feedback intensity.

## Deferred improvements
- Full i18n locale packs for new discovery copy keys in `en`/`nb-NO`.
- Region waitlist/vote CTA backend integration from unavailable state placeholder.
- Additional a11y checks with screen reader snapshots once final copy pipelines land.
